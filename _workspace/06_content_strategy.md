| 기준 | Beginner (초급) | Intermediate (중급) |

| 영어 예상 단어 수 | 4~10 words | 10~20 words |

| hintWords 수 | 3~4개 | 4~6개 |

### 반복 전략
- 각 핵심 단어는 최소 3개, 최대 8개 문장에서 서로 다른 맥락으로 등장
- 하루 3문장 기준, 동일 핵심 단어가 2~3일 간격으로 재등장하도록 출제 알고리즘 설계
- 사용자가 틀린 문장의 핵심 단어는 가중치를 높여 더 자주 노출
- 출제 우선순위: (1) 미학습 단어 > (2) 오답 빈도 높은 단어 > (3) 마지막 노출 후 3일 경과 단어

## 5. DB 스키마 변경 사항

### 기존 Sentence 테이블 확장

```prisma
model Sentence {
  id          String   @id @default(uuid())
  koreanText  String
  theme       String   // "daily" | "business"
  difficulty  String   // "beginner" | "intermediate"
  category    String   // 세부 카테고리 (e.g. "음식/카페", "이메일/서신")
  hintWords   Json     // [{english: string, korean: string}]
  tags        String[] // 핵심 단어 태그 (e.g. ["schedule", "recommend"])
  createdAt   DateTime @default(now())
  corrections Correction[]
}
```

### 새 테이블: CoreWord (핵심 단어 관리)

```prisma
model CoreWord {
  id              String   @id @default(uuid())
  english         String   @unique
  korean          String
  theme           String   // "daily" | "business" | "both"
  targetExposure  Int      // 목표 노출 횟수 (3~8)
  sentenceIds     String[] // 이 단어가 포함된 문장 ID 목록
  createdAt       DateTime @default(now())
}
```

### 새 테이블: UserWordProgress (사용자별 단어 학습 추적)

```prisma
model UserWordProgress {
  id            String   @id @default(uuid())
  userId        String
  coreWordId    String
  exposureCount Int      @default(0)  // 노출 횟수
  correctCount  Int      @default(0)  // 정답 횟수
  lastExposedAt DateTime?
  nextDueAt     DateTime? // 간격 반복 기반 다음 노출 예정일
  user          User     @relation(fields: [userId], references: [id])
  coreWord      CoreWord @relation(fields: [coreWordId], references: [id])

  @@unique([userId, coreWordId])
}
```

### API 변경: GET /api/sentences/daily 출제 알고리즘

기존 랜덤 출제에서 가중치 기반 출제로 변경:

```
1. 미학습 핵심 단어 포함 문장 우선
2. 오답 빈도 높은 핵심 단어 포함 문장 가중치 +2
3. lastExposedAt 기준 3일 이상 경과한 단어 포함 문장 가중치 +1
4. 동일 카테고리 연속 출제 방지 (최근 3일간 출제된 카테고리 제외)
5. 난이도는 사용자 평균 점수 기반 자동 조절 (7점 이상 유지 시 중급 비율 증가)
```
