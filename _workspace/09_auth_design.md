# 쓰잉 v1.1 - 로그인/회원가입 UX 기획

## 현재 상태

- 서버에 이메일+비밀번호 기반 register/login API 존재 (`server/src/routes/auth.ts`)
- 게스트 모드: `optionalAuthMiddleware`에서 토큰 없으면 자동으로 `guest-user-00000000` 할당
- 앱에서는 토큰 저장 인프라 존재 (`expo-secure-store` / `localStorage`)
- 프론트엔드에 로그인/회원가입 화면이 없어 게스트로만 사용 중

## 앱 실행 흐름

```
앱 첫 실행
  │
  ▼
[온보딩 스크린] (3장 스와이프)
  │  "한국어를 영어로 써보세요"
  │  "AI가 교정해 드려요"
  │  "매일 3문장으로 실력 UP"
  │
  ▼
[가입/로그인 선택 스크린]
  ├── [이메일로 시작하기] → 회원가입 스크린
  ├── [이미 계정이 있어요] → 로그인 스크린
  └── [게스트로 둘러보기] → 홈 스크린 (게스트 모드)
```

```
재실행 시 (토큰 존재)
  │
  ▼
토큰 유효성 검증
  ├── 유효 → 홈 스크린
  └── 만료/무효 → 로그인 스크린

재실행 시 (토큰 없음, 게스트 사용 이력 있음)
  │
  ▼
홈 스크린 (게스트 모드 유지)
```

## 화면 설계

### 1. 온보딩 스크린 (OnboardingScreen)

```
┌──────────────────────────────────┐
│                                   │
│         [일러스트/아이콘]          │
│                                   │
│    한국어 문장을 영어로 써보세요    │
│                                   │
│    AI가 자연스러운 표현으로         │
│    교정해 드려요                   │
│                                   │
│          ● ○ ○                    │  ← 페이지 인디케이터
│                                   │
│   ┌──────────────────────────┐    │
│   │       다음                │    │  ← 마지막 장: "시작하기"
│   └──────────────────────────┘    │
│                                   │
│           건너뛰기                 │  ← 텍스트 버튼
│                                   │
└──────────────────────────────────┘
```

- 3장 구성, 좌우 스와이프
- 마지막 장에서 "시작하기" 버튼 → 가입/로그인 선택 스크린으로 이동
- "건너뛰기" 누르면 바로 가입/로그인 선택 스크린

│   └──────────────────────────┘    │
│                                   │

│                                   │
└──────────────────────────────────┘
```

### 3. 회원가입 스크린 (RegisterScreen)

```
┌──────────────────────────────────┐
│  ←                               │
│                                   │
│  회원가입                         │
│                                   │
│  이메일                           │
│  ┌──────────────────────────┐    │
│  │ example@email.com         │    │
│  └──────────────────────────┘    │
│  ※ 유효한 이메일을 입력하세요      │  ← 에러 시 빨간색
│                                   │
│  비밀번호                         │
│  ┌──────────────────────────┐    │
│  │ ●●●●●●●●           👁   │    │  ← 비밀번호 보기 토글
│  └──────────────────────────┘    │
│  ※ 8자 이상 입력하세요            │  ← 실시간 유효성 표시
│                                   │
│  비밀번호 확인                     │
│  ┌──────────────────────────┐    │
│  │ ●●●●●●●●           👁   │    │
│  └──────────────────────────┘    │
│  ※ 비밀번호가 일치하지 않습니다    │
│                                   │
│   ┌──────────────────────────┐    │
│   │       가입하기            │    │  ← 모든 필드 유효 시 활성화
│   └──────────────────────────┘    │
│                                   │
│   이미 계정이 있나요? 로그인       │  ← 텍스트 링크
│                                   │
└──────────────────────────────────┘
```

**유효성 검증 규칙:**
- 이메일: RFC 5322 형식 (`/^\S+@\S+\.\S+$/`)
- 비밀번호: 최소 8자
- 비밀번호 확인: 비밀번호와 일치
- 모든 필드 통과 시 가입 버튼 활성화

**가입 성공 시:**
1. 서버 `POST /api/auth/register` 호출
2. 응답에서 token 수신
3. SecureStore에 token 저장
4. 홈 스크린으로 이동

**에러 처리:**
- 409 (이미 등록된 이메일): "이미 등록된 이메일입니다. 로그인해 주세요."
- 네트워크 오류: "네트워크 연결을 확인해 주세요."

```
┌──────────────────────────────────┐
│  ←                               │
│                                   │
│  로그인                           │
│                                   │
│  이메일                           │
│  ┌──────────────────────────┐    │
│  │ example@email.com         │    │
│  └──────────────────────────┘    │
│                                   │
│  비밀번호                         │
│  ┌──────────────────────────┐    │
│  │ ●●●●●●●●           👁   │    │
│  └──────────────────────────┘    │
│                                   │
│   ┌──────────────────────────┐    │
│   │       로그인              │    │
│   └──────────────────────────┘    │
│                                   │
│   계정이 없나요? 가입하기          │
│                                   │

**로그인 성공 시:**
1. 서버 `POST /api/auth/login` 호출
2. token을 SecureStore에 저장
3. 홈 스크린으로 이동

**에러 처리:**
- 401: "이메일 또는 비밀번호가 올바르지 않습니다."
- 429 (Rate limit): "잠시 후 다시 시도해 주세요."

## 게스트 모드 동작

### 게스트 사용 시 제한사항
| 기능 | 게스트 | 회원 |
|------|--------|------|
| 하루 3문장 학습 | O | O |
| AI 교정 | O | O |
| 학습 기록 보기 | O (기기 내) | O (서버 동기화) |
| XP/레벨/업적 | O (기기 내) | O (서버 동기화) |
| 기기 변경 시 데이터 유지 | X | O |
| 주간 리포트 | O | O |

### 게스트 → 회원 전환 (Account Linking)

설정 화면에 "계정 연동" 버튼을 배치한다.

```
설정 스크린 (게스트 모드일 때)
┌──────────────────────────────────┐
│  설정                             │
│                                   │
│  ┌──────────────────────────┐    │
│  │  계정 연동하기             │    │  ← 강조 표시 (파란색 배경)
│  │  학습 기록을 안전하게 보관   │    │
│  └──────────────────────────┘    │
│                                   │
│  난이도 설정                      │
│  ...                              │
└──────────────────────────────────┘
```

**전환 프로세스:**
1. 사용자가 "계정 연동하기" 탭
2. 이메일 + 비밀번호 입력 폼 표시
3. 서버에 `POST /api/auth/link-guest` 호출
   - 기존 게스트 user 레코드에 email/password 설정
   - 게스트 학습 데이터(corrections, daily_sentences, xp 등)를 새 회원 계정으로 마이그레이션
4. 새 JWT 토큰 발급, SecureStore에 저장
5. "계정 연동 완료!" 토스트 메시지

**필요한 서버 API 추가:**
```
POST /api/auth/link-guest
Body: { email: string, password: string }
Response: { data: { id, email, token } }
```

## authStore 상태 설계

```typescript
interface AuthState {
  isGuest: boolean;
  user: { id: string; email: string } | null;
  token: string | null;
  isOnboarded: boolean;   // AsyncStorage에 영속화
  isLoading: boolean;

  initialize: () => Promise<void>;  // 앱 시작 시 토큰 복원
  login: (email: string, password: string) => Promise<void>;
  register: (email: string, password: string) => Promise<void>;
  linkGuest: (email: string, password: string) => Promise<void>;
  logout: () => Promise<void>;
  skipOnboarding: () => void;
  startAsGuest: () => void;
}
```

## 보안 고려사항

1. **비밀번호 전송**: HTTPS (Railway 기본 제공)
2. **토큰 저장**: iOS/Android는 `expo-secure-store`, 웹은 `localStorage` (현재 구조 유지)
3. **Rate Limiting**: 로그인/가입 엔드포인트에 이미 `authRateLimit` 적용됨
4. **게스트 데이터**: 기기 분실 시 복구 불가 -- 설정 화면에서 주기적으로 "계정 연동" 유도
