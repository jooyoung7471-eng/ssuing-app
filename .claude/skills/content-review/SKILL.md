# Content Review — 문장 DB 품질 검토 및 개선

EngWrite 앱의 학습 콘텐츠 품질을 분석하고 개선한다.

## 검토 대상
- `server/prisma/seed-sentences.json` — 문장 DB
- `server/src/services/llm.ts` — LLM 교정 프롬프트

## 검토 항목

### 1. 문장 자연스러움
- 실제 한국어 일상 대화에서 쓰이는 자연스러운 문장인지
- 교과서적이거나 인위적인 문장은 교체
- 초급: "커피 한 잔 주세요" (O), "나는 커피를 마시기를 원합니다" (X)
- 중급: "요즘 물가가 올라서 외식을 줄이고 있어" (O), "물가 상승으로 인한 외식 빈도 감소" (X)

### 2. 난이도 적절성
초급 기준: 10~20자, 기본 시제, 일상 단어, 단문
중급 기준: 20~40자, 복합 문법, 추상 표현, 관용구, 복합문
난이도가 맞지 않는 문장은 재분류하거나 수정

### 3. 카테고리 균형
각 카테고리별 문장 수 분포 분석. 특정 카테고리에 편중되면 부족한 곳에 추가.

### 4. 핵심 단어 반복
`tags` 필드를 분석하여 핵심 단어의 등장 빈도 확인. 목표: 주요 단어가 3~8회 반복.

### 5. LLM 교정 프롬프트
- 직역체가 나오지 않는지 테스트
- 원어민 표현 제안이 자연스러운지
- 핵심 표현 선정이 학습에 유용한지

## 산출물
- `_workspace/content_report.md` — 분석 결과 + 개선 제안
- 수정된 `seed-sentences.json` (직접 수정 시)
- 수정된 `llm.ts` (프롬프트 개선 시)

## 문장 추가 방법
`claude -p --model haiku`로 배치 생성. 카테고리별 25~50개씩. JSON만 출력하도록 강조.
생성 후 seed-sentences.json에 추가하고 `npx prisma db seed` 재실행.
