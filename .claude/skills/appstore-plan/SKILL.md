# App Store Plan — 앱스토어 메타데이터 기획

EngWrite 영어 작문 학습 앱의 App Store Connect 제출에 필요한 모든 텍스트 메타데이터를 기획한다.

## 앱 컨셉

한글 문장을 보고 영어로 작문하면 AI(LLM)가 교정해주는 학습 앱. 하루 3문장, 일상영어/비지니스영어 테마. 핵심 차별점: AI 실시간 교정 + 집중 학습(하루 3문장) + 게이미피케이션(XP/레벨/업적/스트릭).

## 워크플로우

### Step 1: 앱 현황 파악

1. `_workspace/00_requirements.md` 읽기 — 앱 기능 스펙
2. `src/` 디렉토리 탐색 — 실제 구현된 화면/기능 확인
3. `server/prisma/schema.prisma` — 데이터 모델 확인

### Step 2: 경쟁 앱 분석

한국 앱스토어 교육 카테고리의 상위 영어 학습 앱 벤치마킹:

| 앱 | 참고 포인트 |
|---|---|
| 말해보카 | 앱 이름 패턴, 부제목 구조, 키워드 |
| 듀오링고 | 설명문 구조, 프로모션 텍스트 |
| 케이크 | 스크린샷 카피, 가치 제안 |
| 산타토익 | AI 강조 방식, 키워드 전략 |

WebSearch로 각 앱의 앱스토어 페이지 정보를 수집한다.

### Step 3: 앱 이름 & 부제목

**규칙:**
- 앱 이름: 30자 이내. 기억하기 쉽고, 핵심 키워드 포함
- 부제목: 30자 이내. 앱 이름에 없는 키워드로 보충
- 앱 이름 + 부제목에 같은 키워드 중복 금지 (Apple이 무시함)

**후보 3개를 생성하고 리더에게 선택 요청:**

```
후보 1: [앱 이름] — [부제목]
후보 2: [앱 이름] — [부제목]  
후보 3: [앱 이름] — [부제목]
각 후보의 장단점 + 키워드 커버리지 설명
```

### Step 4: 앱 설명문 (4000자 이내)

**구조:**
```
[첫 3줄 — 핵심 가치 제안. "더 보기" 전에 보이는 부분]

주요 기능
• 기능 1 — 사용자 이점
• 기능 2 — 사용자 이점
• 기능 3 — 사용자 이점

이런 분들에게 추천해요
• 대상 1
• 대상 2

학습 방법
1. 단계 설명
2. 단계 설명
3. 단계 설명
```

### Step 5: 키워드 (100자 이내)

- 쉼표로 구분, 공백 최소화
- 앱 이름/부제목에 이미 있는 단어는 제외
- 검색량 높고 경쟁 낮은 키워드 우선
- 한글 + 영어 키워드 혼합

### Step 6: 기타 메타데이터

| 항목 | 내용 |
|------|------|
| 주 카테고리 | 교육 |
| 부 카테고리 | 생산성 또는 참조 |
| 연령 등급 | 4+ (성인 콘텐츠 없음) |
| 가격 | 무료 |
| 프로모션 텍스트 | 170자 이내. 심사 없이 수시 변경 가능 |
| 릴리스 노트 | 첫 출시 버전 설명 |
| 저작권 | © 2026 [개발자명] |

### Step 7: 개인정보 처리방침

앱이 수집하는 데이터를 코드에서 확인하고 처리방침 초안 작성:
- 수집 데이터: 사용자 작문 내용, 학습 기록, 점수
- 제3자 공유: LLM API 호출 시 작문 내용 전송
- 데이터 보관: 로컬/서버 저장 여부

## App Store Connect 필드 매핑

산출물의 각 항목이 App Store Connect의 어떤 필드에 입력되는지 명시한다:

```
앱 이름          → App Store Connect > 앱 정보 > 이름
부제목           → App Store Connect > 앱 정보 > 부제목  
설명문           → App Store Connect > 버전 정보 > 설명
키워드           → App Store Connect > 앱 정보 > 키워드
프로모션 텍스트   → App Store Connect > 버전 정보 > 프로모션 텍스트
카테고리          → App Store Connect > 앱 정보 > 카테고리
연령 등급         → App Store Connect > 앱 정보 > 연령 등급
```
